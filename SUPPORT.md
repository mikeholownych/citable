# Support

Choose the narrowest channel that fits the request:

| Need | Channel |
| --- | --- |
| Reproducible defect | Bug issue form |
| Suspected false positive or false negative | Detector-result issue form |
| Documentation correction | Documentation issue form |
| Provider or integration proposal | Integration issue form |
| Usage question or troubleshooting | GitHub Discussions: Q&A |
| Product idea not yet accepted for implementation | GitHub Discussions: Ideas |
| Provider research or standards change | GitHub Discussions: Research and standards |
| Security vulnerability | Private vulnerability report on the Security tab |
| Conduct concern | `hello@nebulacomponents.shop` |

## Boundaries

Citable maintainers can support the open-source CLI and its published
contracts. The hosted Nebula Components dashboard, customer accounts, billing,
and service operations belong to the separate hosted product surface.

Do not post credentials, access tokens, cookies, customer data, private URLs,
or unsanitized evidence packages in any public channel. A question without
enough evidence may remain unanswered or be marked `needs-evidence`; that is
not a negative finding about the reporter.

Community support is best-effort. No response time or outcome is guaranteed.
