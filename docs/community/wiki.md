# Wiki Policy

The GitHub Wiki is an operational handbook, not a product contract.

## Appropriate Content

- Getting-started and troubleshooting recipes
- Evidence-package navigation
- Provider integration examples
- Terminology and workflow explanations
- Sanitized community examples and case-study indexes
- Upgrade and migration playbooks that link to released migration notes

## Prohibited As Sole Authority

- Schema definitions or detector requirements
- Security reporting policy
- Supported-version declarations
- Release notes, version numbers, or detector totals
- Governance decisions
- Claims not supported by released artifacts or cited sources

Every page must state `Applies to`, `Last reviewed`, and `Maintainer`, then link
to relevant canonical repository documentation. When Wiki text conflicts with a
released artifact or repository document, the released or repository source
controls.
