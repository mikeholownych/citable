# Discussions Charter

GitHub Discussions is the public space for questions, proposals, provider
research, and examples that are not yet accepted implementation work.

## Categories

- **Announcements:** maintainer-authored project and release notices.
- **Q&A:** usage and troubleshooting; accepted answers are scoped to the stated
  version and evidence.
- **Ideas:** user problems and possible approaches before roadmap acceptance.
- **General with a `[Research]` prefix:** primary-source provider or standards
  changes until a dedicated category can be configured in GitHub's settings UI.
- **Show and tell:** sanitized integrations, workflows, and case evidence.
- **Polls:** exploratory input only; a poll cannot accept work or change policy.

## Promotion To Issues

A maintainer may promote a Discussion when it has a bounded problem, repository
scope, authority source where relevant, acceptance criteria, and identified
evidence limitations. Popularity does not automatically establish priority.

Discussions are non-authoritative. They cannot change schemas, supported
behavior, security policy, release scope, or evidence status.
